# Operación y mantenimiento (día 2)

## Rutina diaria
1. Captura en DB_MASTER_TASKS (fecha, estado, proyecto).
2. Ejecutar COCKPIT: Hoy → Next → Waiting.
3. Cierre: marcar ✅ Done.
4. Métricas (DB_METRICS_LOG) si corresponde.

## Rutina semanal
1. Revisión de proyectos activos (pausar/cerrar).
2. Limpiar backlog → Next.
3. Inbox de lectura: Para Leer → Leyendo/Procesado.
4. Revisión de finanzas.
5. Revisión del silo clínico (de-identificación + permisos).

## Cambios al schema
- Registrar ADR + actualizar manifest.
- Ejecutar repair para reconciliar drift.
